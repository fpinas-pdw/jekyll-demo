Azure ofrece múltiples opciones para ejecutar contenedores. Las dos más populares son Azure Container Instances (ACI) y Azure Kubernetes Service (AKS). Cada una tiene casos de uso ideales y características distintivas.

## Azure Container Instances (ACI)

ACI es la forma más rápida y simple de ejecutar un contenedor en Azure sin necesidad de gestionar infraestructura subyacente.

**Ventajas:**
- Inicio extremadamente rápido (segundos)
- Facturación por segundo
- Sin gestión de infraestructura
- Ideal para workloads temporales

**Casos de uso:**
- Tareas batch y jobs
- CI/CD agents temporales
- Desarrollo y testing
- Aplicaciones simples sin orquestación compleja

## Azure Kubernetes Service (AKS)

AKS es un servicio de Kubernetes gestionado que proporciona orquestación de contenedores de nivel empresarial.

**Ventajas:**
- Orquestación completa
- Escalado automático avanzado
- Service mesh y networking sofisticado
- Ecosistema extenso de herramientas

**Casos de uso:**
- Aplicaciones microservicios complejas
- Workloads de producción a gran escala
- Necesidad de auto-healing y rolling updates
- Integración con GitOps

## Criterios de decisión

Elige **ACI** si:
- Necesitas ejecutar contenedores simples rápidamente
- Tu workload es temporal o estacional
- No requieres orquestación compleja
- Quieres minimizar costos operativos

Elige **AKS** si:
- Tienes arquitecturas microservicios complejas
- Necesitas escalado y auto-healing sofisticados
- Requieres networking avanzado
- Tu equipo tiene experiencia con Kubernetes

No hay una respuesta universal; la mejor elección depende de tus requisitos específicos.
